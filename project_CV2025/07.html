课题07

仅凭视觉输入的游戏自动学习与泛化

返回课题列表

一、背景与动机

在经典的强化学习研究（如 DeepMind DQN）中，智能体（Agent）通常依赖于游戏环境（如 Atari）提供的显式状态（state）信息来学习策略。然而，在真实世界或现代游戏（例如 HTML5 网页游戏、Steam 游戏）中，模型往往无法访问底层状态信息，唯一可见的输入是屏幕画面（pixels）。

本课题的目标是：仅凭视觉输入，让模型自动学习打游戏，并具备跨游戏泛化能力。这不仅是"玩游戏"的问题，更是计算机视觉与智能决策结合的典型任务：如何从视频中提取结构化状态信息，并在视觉世界中实现智能决策？

关键词： 视觉强化学习, 端到端控制, 模仿学习, 多模态大模型, 游戏AI

二、问题与解决方法

1. 缺乏显式 state 与 reward 信号

在无法访问环境底层接口的情况下，智能体无法直接获得"得分"或"状态"。因此需要：

  • 构造状态表示（state representation）
  • 设计代理 reward 或自监督信号

可能方法：

  • 基于视觉特征的状态建模（如特征聚类、关键对象检测、运动估计）
  • 基于变化检测或目标达成检测的 reward 构造（如帧差法、目标检测差异）

Insight：将视觉流转化为可学习的语义信号，是现实场景中通用智能体的第一步。

2. 端到端视觉→动作学习（End-to-End Visual Control）

一种更直接的方式是从像素到动作的端到端学习，跳过显式状态建模环节。

方法1：模仿学习（Behavioral Cloning, BC）

利用人类游戏演示数据（视频帧 + 动作键位），训练模型模仿人类行为。

  • 思想：模仿而非强化，先从已有的专家经验出发。
  • 模型结构：CNN / Transformer 提取视觉特征 → MLP 输出离散动作
  • 优点：简单、稳定，适合初学者实现。

开源参考：

  • DeepLearningFlappyBird
  • Behavioral Cloning with Gym
  • PyBoy Pokémon BC Demo

方法2：大模型推理式控制（Multimodal LLM-based Control）

利用多模态大模型（如 Gemini、GPT-4V、Voyager、GATO）理解屏幕画面并生成动作。

  • 输入：当前游戏截图 + 文本提示（例如"下一步怎么移动？"）
  • 输出：键盘 / 鼠标动作（或高层自然语言指令）

思路：通过大模型的视觉理解和语言推理能力实现"基于推理的游戏控制"。

相关参考：

  • VG-Bench
  • Gemini Plays Pokémon

3. 复杂动作空间设计（Action Space Design）

在鼠标类或高自由度的游戏中，动作不仅是离散按键，而是连续的坐标（如鼠标点击、拖拽等）。直接输出坐标可能导致不稳定或无意义的动作。

可行方案：

  • 使用 动作离散化（Discretization）：将屏幕划分为网格区域
  • 或基于 目标检测/注意力机制：预测下一步应关注的视觉区域
  • 可结合工具：
    • pyautogui — 键鼠模拟
    • gymnasium-robotics — 高维动作建模

4. 游戏表现评估指标（Evaluation Metrics）

在没有标准得分接口的视觉游戏中，评估模型表现是一大挑战。可考虑以下几类指标：

类型 说明 示例
基于目标的成功率 模型是否达成特定目标（如通关、存活时间） 通过画面检测"Game Over"或"Next Level"文字
基于动作多样性 模型是否能产生丰富、合理的动作序列 动作熵、去重率
基于轨迹一致性 模型是否能生成与人类轨迹一致的视觉流 SSIM、FID、动态特征差距
基于视觉奖励估计 基于视频变化推测的代理 reward 帧差、运动量、关键对象得分变化
跨任务泛化性能 在新游戏中能否迁移已有策略 Few-shot 适应能力测试
